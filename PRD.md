## 1. 文件目的

本文件定義「靈感樹｜AI 心智圖工作室」的產品目標、使用情境、功能需求、驗收條件、技術限制與後續路線，作為產品設計、前端開發、後端整合、測試與後續 AI 開發的共同依據。

## 2. 產品摘要

靈感樹是一款以視覺化放射思考為核心的心智圖工具。使用者可在溫暖、低干擾的畫布上新增、編輯、拖曳與移除節點，透過漸細貝茲曲線理解父子關係，並從右側 AI 思考夥伴取得延伸方向。

目前版本定位為公開 Beta：支援本機自動儲存、可編輯文件標題、共享地圖、版本衝突提示、畫布與大綱雙模式、鍵盤操作、AI 建議批次預覽，以及 PNG、PDF、Markdown 匯出。AI 建議仍為前端內建情境內容，尚未串接實際大型語言模型。

## 3. 問題與機會

### 3.1 使用者問題

- 想法分散在筆記、聊天與腦中，缺乏清楚的結構。
- 傳統心智圖工具操作負擔較重，容易打斷思考節奏。
- 使用者在卡住時，缺乏能依目前思路提供下一步的引導。
- 完成的心智圖不易快速轉成可分享的圖片、文件或文字大綱。

### 3.2 產品機會

- 以低學習成本的視覺工作室降低整理想法的門檻。
- 將 AI 建議放在目前聚焦節點的情境中，而非提供泛用聊天框。
- 同時支援視覺輸出與結構化文字輸出，銜接簡報、文件及工作協作。

## 4. 產品目標與非目標

### 4.1 產品目標

1. 使用者能在 30 秒內開始建立或延伸心智圖。
2. 使用者能清楚辨識中心、主要分支與次要分支。
3. 所有核心編輯操作均可復原與重做，降低操作焦慮。
4. 使用者能將成果匯出為 PNG、PDF 或 Markdown。
5. 未來 AI 服務能依目前節點、祖先路徑及使用者指令產生結構化建議。

### 4.2 非目標

- 現階段不取代完整專案管理、白板或流程圖產品。
- 現階段不支援多人即時游標與同步編輯。
- 現階段共享地圖採儲存後同步，不提供即時多人游標。
- 現階段 AI 建議不代表真實模型推論結果。

## 5. 目標使用者

### 5.1 主要使用者

- 知識工作者：整理專案、會議、研究與決策脈絡。
- 創作者：發想內容、故事、課程與活動企劃。
- 學生與教師：拆解主題、建立學習架構與教學大綱。
- 個人成長使用者：規劃生活目標、習慣與行動方案。

### 5.2 核心工作

- 「當我有一個模糊想法時，我希望快速拆成可理解的分支。」
- 「當我卡住時，我希望得到貼近目前思路的延伸建議。」
- 「當我完成整理後，我希望能輸出成圖片、文件或文字大綱。」

## 6. 核心使用流程

### 6.1 建立與延伸

1. 使用者進入首頁並看到預設心智圖，可修改正上方文件標題。
2. 使用者點選欲延伸的節點。
3. 使用者從左側工具列、節點快捷按鈕或 AI 建議新增子節點。
4. 新節點成為目前聚焦節點，並顯示對應 AI 建議。

### 6.2 編輯與整理

1. 使用者雙擊節點或點擊編輯按鈕，就地修改標題與補充說明。
2. 使用者拖曳節點調整空間位置。
3. 連接線即時跟隨節點，維持父子關係。
4. 使用者可復原或重做新增、移除、編輯及拖曳操作。
5. 使用者可切換大綱模式、搜尋節點，或收合指定分支。

### 6.3 移除

1. 使用者選取非中心節點。
2. 點擊左側「移除」。
3. 系統移除該節點及其所有子節點。
4. 焦點回到被移除節點的父節點。
5. 使用者可透過「復原」還原整個分支。

### 6.4 匯出

1. 使用者點擊右上角「匯出」。
2. 選擇 Markdown、PDF 或 PNG。
3. 系統使用中心節點名稱產生安全檔名並開始下載。

## 8. 資料模型

### 8.1 目前前端節點

| 欄位 | 型別 | 說明 |
|---|---|---|
| id | number | 節點唯一識別 |
| parent | number \| null | 父節點；中心節點為 null |
| text | string | 節點標題 |
| note | string | 補充說明 |
| x / y | number | 畫布位置 |
| tone | enum | ink、coral、sage、sun |

### 8.2 雲端版建議模型

- MindMap：id、title、data、version、updatedAt、updatedBy；目前以單一 JSON 保存節點圖。
- MindMapNode：id、mindMapId、parentId、text、note、x、y、tone、order。
- ShareLink：id、mindMapId、permission、expiresAt。
- AIRequest：id、mindMapId、nodeId、prompt、status、createdAt。

## 9. 非功能需求

### 9.1 效能

- 一般桌面裝置首次互動時間目標低於 2.5 秒。
- 100 個節點時拖曳應維持可用，不出現明顯停頓。
- 連接線與節點更新不得觸發不必要的全頁重新載入。

### 9.2 可用性與無障礙

- 互動按鈕必須具備可理解的無障礙名稱。
- 停用狀態需同時以視覺與原生 disabled 表達。
- 支援系統「降低動態效果」偏好。
- 文字與背景應符合 WCAG AA 對比要求。
- 已支援 Enter 新增同層、Tab 新增子節點、Delete/Backspace 移除及 Esc 取消編輯。

### 9.3 響應式

- 桌面版完整顯示畫布與 AI 面板。
- 平板與窄螢幕不得遮蔽核心新增、移除與匯出流程。
- 手機版 AI 面板採可收合 bottom sheet，預設保留畫布操作空間。
- 觸控拖曳需避免與頁面捲動互相衝突。

### 9.4 安全與隱私

- AI 金鑰與第三方憑證只能存在伺服器端。
- 使用者內容不得未經同意用於公開分享。
- 分享連結需具備撤銷、到期與唯讀權限。
- 雲端資料需依使用者身分隔離。

## 10. 成功指標

### 10.1 啟用

- 首次工作階段建立至少一個新節點的使用者比例。
- 從載入到第一次有效操作的中位時間。

### 10.2 使用深度

- 每份心智圖平均節點數與最大深度。
- AI 建議被加入心智圖的比例。
- 復原／重做使用率及移除後復原比例。

### 10.3 產出

- 完成至少一次匯出的工作階段比例。
- PNG、PDF、Markdown 的格式使用分布。
- 建立後七日內再次開啟或編輯的比例。

## 13. 風險與對策

| 風險 | 影響 | 對策 |
|---|---|---|
| 大型心智圖造成重新渲染負擔 | 拖曳卡頓 | 拆分節點元件、限制更新範圍、必要時採虛擬化 |
| AI 輸出格式不穩定 | 無法建立節點 | 使用結構化輸出、伺服器驗證與重試 |
| 自動布局破壞手動位置 | 使用者失去控制感 | 提供預覽、復原與局部布局 |
| 分享連結洩漏私人內容 | 隱私風險 | 預設私人、明確權限、到期與撤銷 |
| 匯出字型差異 | 圖片或 PDF 排版不一致 | 等待字型載入、設定備援字型、建立跨瀏覽器測試 |

## 15. 完成定義

單一功能只有在以下條件全部成立時才視為完成：

1. 功能符合本文件的驗收條件。
2. 涵蓋正常、停用、空資料與錯誤狀態。
3. 具備必要的無障礙名稱與鍵盤操作。
4. 通過正式建置及對應互動測試。
5. 若影響匯出或保存，需同步驗證輸出及資料還原。
6. 已更新 PRD、README 或開發紀錄中的相關說明。
